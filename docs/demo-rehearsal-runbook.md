# Demo Rehearsal Runbook

Step-by-step commands for running the full demo in your own terminal.
Copy-paste each block, observe the output, then move to the next step.

---

## Prerequisites

- GKE cluster running (`./demo/cluster/setup.sh gcp`)
- `demo/.env` generated by setup

## Reset (between runs)

```bash
./demo/cluster/reset-demo.sh
```

Confirm: one pod in CrashLoopBackOff, no ManagedService claims, no thread checkpoints.

---

## Step 1: Source environment

```bash
source demo/.env
```

This sets infrastructure URLs only. No audience-facing vars yet.

## Step 2: Verify no kubectl access

```bash
kubectl get pods
```

Expected: `error: no cluster configured` (presenter's shell has no kubeconfig).

---

## Step 3: Act 2 — Agent investigates with kubectl

Pick an agent framework (langgraph or vercel):

```bash
export CLUSTER_WHISPERER_AGENT=langgraph
export CLUSTER_WHISPERER_TOOLS=kubectl
```

```bash
cluster-whisperer "Something's wrong with my application — can you investigate what's happening and why?"
```

Watch for: agent uses kubectl_get, kubectl_describe, kubectl_logs. Finds CrashLoopBackOff, missing database.

```bash
cluster-whisperer "Do you know what database I should use?"
```

Watch for: agent tries kubectl get crd, sees hundreds of CRDs, can't make sense of them without semantic search.

---

## Step 4: Act 3a — Add vector search (multi-turn)

Pick a vector backend (chroma or qdrant):

```bash
export CLUSTER_WHISPERER_VECTOR_BACKEND=qdrant
export CLUSTER_WHISPERER_TOOLS=kubectl,vector
export CLUSTER_WHISPERER_THREAD=demo
```

**Turn 1:**

```bash
cluster-whisperer "What database should I deploy for my app?"
```

Watch for: agent uses vector_search on capabilities collection. Finds 20 ManagedService resources from different teams. Asks follow-up questions.

**Turn 2:**

```bash
cluster-whisperer "I'm not sure about most of that. My team is called the You Choose team. I don't know if it's Postgres or MySQL."
```

Watch for: agent searches again with "You Choose" context. Narrows to `managedservices.platform.acme.io`. Recommends it with example YAML.

**Turn 3:**

```bash
cluster-whisperer "Yes please, will you deploy it for me?"
```

Watch for: agent says it **cannot deploy** — no apply tool. Provides YAML for manual use.

---

## Step 5: Act 3b — Deploy

Add the apply tool:

```bash
export CLUSTER_WHISPERER_TOOLS=kubectl,vector,apply
```

```bash
cluster-whisperer "Go ahead and deploy it"
```

Watch for: agent uses kubectl_apply with the ManagedService YAML. Reports SYNCED and READY.

**Wait ~15 seconds** while talking through what Crossplane is doing (provisioning PostgreSQL, creating db-service).

```bash
cluster-whisperer "Is my app running now? What's the URL to access it?"
```

Watch for: agent checks pods, finds demo-app Running (or about to be). Checks ingresses, returns `http://demo-app.<base-domain>`.

**Open the URL in a browser.** Spider page with YouTube links should appear.

If the app is still in CrashLoopBackOff (backoff delay), wait 30 seconds and ask again — Kubernetes will restart the pod and it will connect.

---

## Step 6: Act 4 — Observability

Open whichever backend you want:

- **Jaeger**: `http://jaeger.<base-domain>` — search for service `cluster-whisperer`
- **Datadog**: Datadog LLM Observability UI — search for traces

Everything was instrumented with OpenTelemetry the whole time.

---

## Troubleshooting

**Vector search returns "not available":**
Check that `CLUSTER_WHISPERER_QDRANT_URL` (or `_CHROMA_URL`) is exported. `source demo/.env` sets it but doesn't export. Run: `export CLUSTER_WHISPERER_QDRANT_URL`

**Agent says "database already deployed":**
Stale thread memory. Run `rm -f data/threads/*.json` and start Act 3a over.

**App stays in CrashLoopBackOff after deploy:**
The CrashLoopBackOff backoff delay increases with each restart. Either wait (Kubernetes will retry) or delete the pod: `KUBECONFIG=~/.kube/config-cluster-whisperer kubectl delete pod -l app=demo-app`

**Agent hits recursion limit (50 steps):**
The agent is trying to fix the app instead of waiting for Crossplane. Split the deploy and URL check into separate prompts with a pause between.

**Spider page shows JSON instead of image:**
Old Docker image on the cluster. Rebuild: `docker buildx build --platform linux/amd64 --provenance=false -t wiggitywhitney/demo-app:latest --push demo/app` then delete the pod.
